Lịch sử Cơ sở Dữ liệu

Trước 1960
Quản lý thủ công
  • Lưu trữ: Giấy, thẻ đục lỗ, băng từ
  • Hạn chế: Tốn thời gian, dễ sai sót
1960s
Hệ thống tệp & Mô hình phân cấp
  • Tệp-based systems: Ứng dụng tự quản lý dữ liệu
  • IMS (IBM): Cấu trúc cây
  • Hạn chế: Thiếu tính độc lập dữ liệu
1970s
Mô hình mạng & Quan hệ
  • Mô hình mạng: CODASYL, cấu trúc đồ thị
  • Mô hình quan hệ (E.F. Codd)
    • Tổ chức: Bảng, khóa
    • SQL: Ngôn ngữ truy vấn
    • DBMS: System R, Ingres
1980s
DBMS quan hệ bùng nổ
  • Thương mại: Oracle, DB2, Sybase
  • Cải tiến: Giao diện đồ họa, tối ưu truy vấn
  • Ứng dụng: Ngân hàng, hàng không
1990s
Hướng đối tượng & Kho dữ liệu
  • OODBMS: ObjectStore, GemStone
  • Data Warehouse: BI, OLAP
  • Web: Tích hợp dữ liệu trực tuyến
2000s
NoSQL & Big Data
  • NoSQL
    • Loại: MongoDB, Cassandra, Redis, Neo4j
    • Đặc điểm: Phi cấu trúc, mở rộng ngang
  • Big Data: Hadoop, Spark
2010s - Nay
Đám mây & AI
  • Cloud DB: AWS RDS, Google Spanner
  • Thời gian thực: Kafka, DynamoDB
  • AI/ML: Phân tích dự đoán
  • NewSQL: CockroachDB, TiDB
  • Blockchain: CSDL phi tập trung
Tương lai
  • Đa mô hình: PostgreSQL, MongoDB
  • Tự động hóa: AI tối ưu hóa
  • Bảo mật: GDPR, mã hóa
  • CSDL lượng tử
